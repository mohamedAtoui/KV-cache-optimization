# Attention Mechanism Comparison: MHA vs MQA vs GQA

A research project comparing three attention mechanism architectures in transformer-based language models, trained on the TinyStories dataset.

## Overview

This project implements and compares:

| Architecture | Description | Key Trade-off |
|-------------|-------------|---------------|
| **MHA** (Multi-Head Attention) | Baseline with separate K/V heads per query head | Best quality, highest memory |
| **MQA** (Multi-Query Attention) | Single shared K/V head across all query heads | Fastest inference, some quality loss |
| **GQA** (Grouped Query Attention) | Groups of query heads share K/V heads | Best balance of quality and efficiency |

## Model Configuration

All models share the same base architecture:

- **Architecture**: Decoder-only transformer (GPT-style)
- **Hidden Size**: 256
- **Layers**: 4
- **Attention Heads**: 8
- **Max Sequence Length**: 256 tokens
- **Parameters**: ~10-16M (varies by attention type)

## Dataset

**TinyStories** - A dataset of 2.1M synthetic children's stories generated by GPT-3.5/GPT-4. Selected for:
- Simple vocabulary (~10K words)
- Clean sentence structure
- Fast convergence on small models

## Results

| Model | Parameters | Validation Perplexity | KV-Cache Size |
|-------|------------|----------------------|---------------|
| MHA   | ~16M       | ~36-40               | 1x (baseline) |
| MQA   | ~12M       | ~45-55               | 0.125x        |
| GQA-2 | ~9.8M      | ~45-55               | 0.25x         |

## Usage

### Training

```bash
cd AttentionHeads
pip install -r requirements.txt

# Train MHA baseline
python -m mha.train --config mha/config.json

# Train MQA variant
python -m mqa.train --config mqa/config.json

# Train GQA variant
python -m gqa.train --config gqa/config_kv2.json
```

### Notebooks

Training and evaluation notebooks are in `AttentionHeads/notebooks/`:
- `train_gptneo_tinystories.ipynb` - MHA training
- `train_gptneo_mqa_tinystories.ipynb` - MQA training
- `train_gptneo_gqa4_tinystories.ipynb` - GQA training
- `compare_mha_mqa_models.ipynb` - Model comparison
- `evaluate_attention_comparison.ipynb` - Full evaluation

## Key Findings

1. **MQA** reduces attention parameters by 44% and KV-cache by 8x, with acceptable quality degradation
2. **GQA** provides the best trade-off between quality and efficiency
3. GQA is used in production models like LLaMA 2 and Mistral 7B for this reason
